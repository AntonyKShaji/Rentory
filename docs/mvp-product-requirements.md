# Rentory MVP Product Requirements

## 1. Objective
Build a mobile MVP for owners and tenants to manage rent operations, bill tracking, communication, and maintenance workflows.

## 2. User roles

### Owner
- Manage multiple properties grouped by location.
- View occupancy and tenant details.
- Track rent, electricity, and water bill statuses.
- Send notifications and announcements.
- Review transaction history and lease validity.
- Assign service personnel (plumber, electrician, supervisor).

### Tenant
- Join property via QR onboarding request.
- View assigned property dashboard.
- Pay rent and utility dues.
- Raise maintenance/service requests.
- Receive owner announcements and reminders.

## 3. Core MVP features

1. Authentication (phone/email OTP, role-based profile).
2. Property management (owner CRUD: location, unit details, capacity).
3. Tenant mapping (owner approves tenant join requests).
4. Bill status tracking (Rent / E-Bill / Water).
5. Payment records and downloadable receipts.
6. Notifications (due reminders, custom announcements).
7. Maintenance request pipeline from tenant to owner.
8. Basic analytics (income, expenses, net monthly summary).

## 4. Non-MVP (future)

- In-app tenant group split expense workflow.
- Full chat/messaging history.
- Automated smart collections and retries.
- AI-based rent forecasting and vacancy suggestions.

## 5. Subscription concept

### Free trial
- Up to 1 property.
- Limited notification broadcasts.

### Pro Owner
- Multiple properties.
- Analytics and exports.
- Priority service request workflow.

### Enterprise
- Unlimited properties.
- Advanced roles (supervisor, accountant).
- API access and deeper integrations.
